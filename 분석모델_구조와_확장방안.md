# WK리그 마케팅 인사이트 분석 모델 — 구조와 확장 방안

> 이 문서는 분석 모델이 **무엇을 하고**, **어떤 구조이며**, **데이터가 추가되면 어떻게 확장되는지**를 정리한 것이다. 공모전 제안서의 "지속 가능한 운영 체계" 파트에 활용할 수 있다.

---

## 1. 이 모델은 무엇인가

**한 줄 정의** — 공개 데이터로 WK리그의 관심층을 측정하고, 데이터에 근거해 다음 타겟층과 실행 액션을 자동으로 도출하는 **반복 가능한 분석 시스템**.

**핵심 아이디어** — 리그 활성화는 한 번의 캠페인으로 끝나지 않는다. "측정 → 인사이트 도출 → 실행 → 재측정"이 계속 돌아야 한다. 이 모델은 그 사이클을 **사람이 매번 수작업하지 않도록** 자동화한 것이다.

- 단발성 분석 보고서가 아니라, **언제든 다시 돌릴 수 있는 시스템**
- 분석 → 인사이트 → 리포트까지 자동, 사람은 **'실행 판단'에만 집중**

---

## 2. 전체 구조 — 3계층

```
┌─────────────────────────────────────────────┐
│  1. 데이터 수집층  (collectors.py)            │
│     외부 소스에서 원시 데이터를 가져온다        │
└───────────────────┬─────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  2. 분석층  (analyzers.py)                    │
│     원시 데이터를 인사이트 가능한 지표로 가공    │
└───────────────────┬─────────────────────────┘
                    ↓
┌─────────────────────────────────────────────┐
│  3. 인사이트 산출층  (insights.py + app.py)    │
│     AI가 자연어 리포트 생성 + 대시보드 시각화   │
└─────────────────────────────────────────────┘
```

이 3계층 구조의 장점은 **수집층만 바꾸면 나머지는 그대로 확장된다**는 것이다. 새 데이터원(구단 자료, 인스타 등)이 들어와도 분석층·인사이트층의 틀은 유지된다. (→ 7장 참고)

---

## 3. 현재 기능 (공개 데이터 버전)

현재는 **누구나 접근 가능한 공개 데이터**만 사용한다. 5개 기능으로 구성된다.

| # | 기능 | 사용 데이터 | 산출물 | 현재 한계 |
|---|------|-----------|--------|----------|
| 1 | 검색 트렌드 분석 | 네이버 검색 인덱스 | 관심도 추이, 요일 패턴, 작년 동기 대비 | 절대 규모는 알 수 없음(상대값) |
| 2 | 변화·스파이크 감지 | 네이버 검색 + 뉴스 | 급증 자동 탐지 + 원인 추정 | 원인은 뉴스 기반 '추정' |
| 3 | 관심층 인구통계 | 네이버 검색 세그먼트 | 연령·성별 분포 | '검색층'이지 '실제 관람객'은 아님 |
| 4 | 유튜브 콘텐츠 반응 | YouTube 공개 통계 | 포맷별 조회수·참여율 | 시청자가 누구인지는 모름 |
| 5 | AI 인사이트 리포트 | 위 1~4 종합 | 다음 타겟 + 실행 액션 | 관심 단계까지만 분석 |

**현재 모델이 측정하는 범위** — "사람들이 WK리그에 **관심을 보이는가**"까지. 그 관심이 실제 직관·구매로 **이어졌는가**는 아직 측정하지 못한다. 그 이유는 그 데이터(관중·티켓·SNS 내부 지표)가 연맹·구단만 가지고 있기 때문이다.

---

## 4. 이 모델이 주는 도움

1. **추측이 아닌 데이터로 타겟을 정한다.** "20대 여성이 많을 것 같다"가 아니라 검색 데이터로 실제 관심층을 확인한다.
2. **캠페인 효과를 검증한다.** 캠페인 전후의 검색량·인구통계 변화를 비교해 "효과가 있었는가"를 숫자로 본다.
3. **무엇이 먹히는지 학습한다.** 검색 스파이크의 원인을 자동 포착해, 어떤 이벤트·콘텐츠가 관심을 끌었는지 축적한다.
4. **반복 가능하다.** 매주/매월 다시 돌리면 팬층 변화를 추적하는 중장기 운영 체계가 된다.
5. **분석 인력 부담을 줄인다.** AI가 리포트를 자동 작성하므로, 담당자는 해석과 실행에만 집중한다.

## 5. 데이터가 추가되면 — 확장 시나리오

> 아래는 **연맹·구단·계정 소유자의 협조로 추가 데이터를 얻을 수 있다고 가정**했을 때, 모델이 어떻게 확장되는지를 정리한 것이다.

### 5-1. 구단 관중·티켓 데이터가 추가되면

| 구분 | 현재 | 데이터 추가 시 |
|------|------|--------------|
| 측정 범위 | 검색량(관심)까지 | **실제 유입(관중 수)까지** |
| 핵심 신규 지표 | — | **전환율 = 관중 ÷ 관심**, 재방문율, 좌석 점유율 |
| 신규 분석 | — | 구단별·지역별 비교, 경기별 흥행 요인, 티켓 가격 민감도 |

**무엇이 가능해지나** — 지금은 "검색이 늘었다"까지만 보지만, 관중 데이터가 붙으면 "검색이 는 만큼 **실제로 경기장에 왔는가**"를 계산할 수 있다. 관심은 높은데 직관으로 안 이어지는 '전환 누수 구간'을 찾아낼 수 있다.

### 5-2. 인스타그램 계정주 데이터가 추가되면

> 인스타그램 팔로워의 연령·성별·지역 같은 상세 인구통계는 **계정 소유자만** 볼 수 있다. 연맹·구단 공식 계정의 데이터를 공유받아야 한다.

| 구분 | 현재 | 데이터 추가 시 |
|------|------|--------------|
| 인구통계 | 네이버 검색 세그먼트(프록시) | **실제 팔로워 인구통계** (연령·성별·지역) |
| 콘텐츠 분석 | 유튜브 공개 통계만 | 게시물별 **도달·저장·공유·프로필 방문** |
| 유입 경로 | — | 게시물 → 프로필 → 외부 링크 클릭까지 추적 |

**무엇이 가능해지나** — "검색으로 드러난 관심층"과 "실제로 팔로우한 사람"을 **비교**할 수 있다. 둘이 다르면(예: 검색은 10대가 많은데 팔로워는 30대가 많다면) 어느 채널이 어떤 층을 데려오는지가 보인다. 또 어떤 게시물이 신규 팔로워·유입을 만드는지 콘텐츠별로 측정된다.

### 5-3. 유튜브 채널 소유자 데이터(YouTube Analytics)가 추가되면

| 구분 | 현재 | 데이터 추가 시 |
|------|------|--------------|
| 영상 분석 | 조회수·좋아요·댓글(공개) | **시청자 인구통계, 평균 시청 지속시간, 트래픽 소스** |
| 전환 측정 | — | 영상 → 구독 전환율, 영상 → 채널 방문 |

**무엇이 가능해지나** — 지금은 "풀경기 영상이 조회수가 높다"까지만 안다. 소유자 데이터가 붙으면 "**어떤 영상이 구독자(=지속 팬)를 만드는가**", "시청자가 끝까지 보는가"를 알 수 있어, 단순 조회수가 아닌 '팬 전환 기여도'로 콘텐츠를 평가할 수 있다.

### 5-4. 설문·인터뷰 데이터가 추가되면

- 검색·관중 데이터는 "무엇을(what)" 보여주지만 "왜(why)"는 못 보여준다.
- 설문·인터뷰 같은 **정성 데이터**가 붙으면, 숫자 뒤의 동기("왜 직관을 안 가는가", "무엇이 있으면 가겠는가")를 해석에 결합할 수 있다.
- AI 리포트가 정량 지표와 정성 응답을 **함께 종합**해 더 설득력 있는 인사이트를 낸다.

### 5-5. 웹사이트 트래픽 데이터(구글 애널리틱스 등)가 추가되면

- 공식 사이트·예매 페이지의 유입 경로와 이탈 지점을 본다.
- "SNS에서 사이트로 왔는데 예매 직전에 이탈한다" 같은 **퍼널 누수 지점**을 정확히 짚어낸다.

## 6. 최종 지향 — 관심→유입→전환 풀 퍼널

마케팅 퍼널 전체에서 **현재 모델이 보는 범위**와 **데이터 추가 시 보게 될 범위**:

```
   인지        관심        유입         전환          충성
(노출·검색) (검색·팔로우) (시청·방문)  (직관·구매)  (재방문·시즌권)
 │          │           │            │             │
 ├──── 현재 측정 범위 ────┤            │             │
 │   (공개 데이터)        │            │             │
 │                                                  │
 ├──────────── 데이터 추가 시 측정 범위 ──────────────┤
     (구단·인스타·유튜브 소유자·설문 데이터 연동)
```

- **현재** — 인지~관심 단계 측정. "관심이 생겼는가"까지.
- **확장 후** — 유입~충성 단계까지 측정. "관심이 직관·재방문으로 이어졌는가"까지.

즉 현재 모델은 **완성된 퍼널의 앞부분**을 담당하며, 추가 데이터가 들어오면 **같은 구조 위에서 뒷부분으로 자연스럽게 확장**된다.

---

## 7. 기술적으로 어떻게 확장되는가

새 데이터가 추가될 때 **시스템 전체를 다시 만들 필요가 없다.** 모듈식 구조이기 때문이다.

| 계층 | 새 데이터 추가 시 | 작업량 |
|------|-----------------|-------|
| 1. 수집층 | 새 데이터원 **커넥터 하나만 추가** (예: 구단 티켓 API) | 작음 |
| 2. 분석층 | 새 지표 함수 추가 (전환율 등). 기존 분석은 유지 | 중간 |
| 3. 인사이트층 | AI 프롬프트에 새 지표 항목 추가 | 작음 |
| 대시보드 | 새 섹션/탭 추가 | 중간 |

**핵심** — 수집층은 "외부 데이터 → 표준 형식"으로 바꿔주는 어댑터 역할을 한다. 데이터원이 늘어도 분석층·인사이트층은 표준 형식만 받으면 되므로, **확장 비용이 선형적으로 작게 유지**된다.

---

## 8. 요약

- 이 모델은 **공개 데이터로 WK리그 관심층을 측정하는 반복 가능한 시스템**이며, 현재 마케팅 퍼널의 '인지~관심' 단계를 담당한다.
- 구단·인스타·유튜브 소유자·설문 데이터가 추가되면 **'유입~전환~충성' 단계까지** 같은 구조 위에서 확장된다.
- 확장 시 핵심 변화는 **"관심이 실제 행동으로 이어졌는가"를 측정**하게 되는 것 — 전환율, 유입 경로, 콘텐츠별 팬 전환 기여도 등.
- 3계층 모듈 구조라 데이터가 늘어도 시스템을 재설계할 필요 없이 **커넥터만 추가**하면 된다.

> **제안서 활용 포인트** — "우리는 한 번 쓰고 끝나는 분석이 아니라, 데이터가 쌓일수록 더 정교해지는 **성장하는 시스템**을 제안한다. 지금은 공개 데이터로 시작하지만, 연맹·구단의 협조로 데이터가 연동될수록 관심에서 전환까지 전체 여정을 추적하게 된다."

# WK리그 마케팅 인사이트 대시보드

공개 데이터만으로 WK리그(여자축구)의 관심층을 추적하고, 데이터 기반으로 다음 타겟 세그먼트와 실행 액션을 자동 도출하는 대시보드.

공모전 시연용으로 설계되었으며, API 키가 없어도 mock 데이터로 전체 흐름을 확인할 수 있다.

---

## 빠른 시작 (API 키 없이도 가능)

```bash
# 1. 의존성 설치
pip3 install -r requirements.txt

# 2. 대시보드 실행
streamlit run app.py
```

브라우저에서 `http://localhost:8501` 열림.
명령어가 안 먹히면 `python3 -m streamlit run app.py` 로 실행.

---

## 실제 데이터로 돌리려면 (API 키 3개)

### 1. 네이버 개발자 센터 (검색 트렌드 + 뉴스)

1. https://developers.naver.com 가입
2. Application 등록 → 사용 API: **데이터랩(검색어 트렌드)** + **검색** 둘 다 체크
3. 비로그인 오픈 API 서비스 환경: **WEB 설정** → `http://localhost:8501`
4. Client ID, Client Secret 복사

### 2. YouTube Data API v3

1. https://console.cloud.google.com → 프로젝트 생성
2. API 라이브러리에서 **YouTube Data API v3** 활성화
3. 사용자 인증 정보 → **공개 데이터** 선택 → API 키 발급

### 3. Anthropic API (Claude)

1. https://console.anthropic.com 가입 → 결제수단 등록
2. API 키 발급 (시연 수준이면 5,000~10,000원이면 충분)

### 4. 빅카인즈 (선택, 권장 — 뉴스 표본 ~10배 확대)

1. https://www.bigkinds.or.kr 가입 → 마이페이지 → API 신청
2. 발급된 ACCESS_KEY를 `.env`의 `BIGKINDS_ACCESS_KEY`에 입력
3. 미설정 시 mock 폴백으로 작동

> 구글 트렌즈·위키피디아 페이지뷰는 **키 없이** 자동으로 작동.

### 5. 키 설정

```bash
cp .env.example .env
# .env 파일을 편집해서 발급받은 키 채우기
```

API 키별로 독립적으로 작동한다 — 네이버만 넣으면 검색·뉴스만 실데이터, 나머지는 mock.

## 대시보드 섹션 안내

1. **검색 트렌드** — WK리그 검색 인덱스 추이, 요일별 패턴, 작년 동기 대비(YoY).
2. **키워드 세분화 & 국대 보정** — 'WK리그' 관심을 구단·선수·국가대표로 분리. 국가대표 검색과 함께 움직이는 부분을 제거한 '순수 WK리그 관심도' 산출.
3. **변화 감지 및 스파이크** — 최근 14일 성장률 + 검색량 급증 자동 탐지. 각 스파이크를 같은 시점 뉴스와 연결해 원인·유형을 자동 분류.
4. **관심층 인구통계 & 변화** — 검색층 연령·성별 분포 + 기간에 따른 관심층 이동(상승/하락 세그먼트).
5. **유튜브 콘텐츠 반응** — 공식 채널 영상 포맷별(풀경기·하이라이트·골모음 등) 평균 조회수·참여율.
6. **여론 감성 분석** — 유튜브 댓글의 긍·부정 분류. 관심의 '양'이 아닌 '질'을 측정.
7. **경기력 ↔ 관심도** — 중계 여부·접전 여부가 검색 관심에 주는 영향. (data/match_records.csv 기반)
8. **AI 자동 인사이트 리포트** — Claude가 위 1~7번을 종합해 "현재 관심층 / 변화 / 다음 타겟 / 실행 액션" 자동 생성.

### 사용 데이터 소스 (총 7종)

| 소스 | 키 | 역할 |
|---|---|---|
| 네이버 데이터랩 검색어 트렌드 | 필요 | 검색 인덱스 + 인구통계 |
| 네이버 검색 API (뉴스·블로그·카페) | 필요 | 뉴스 헤드라인 + 커뮤니티 언급 + 감성 표본 |
| YouTube Data API v3 | 필요 | KWFF 채널 + 외부 채널 영상·댓글 |
| 구글 트렌즈 (pytrends) | 불필요 | 네이버와 cross-validation |
| 위키피디아 페이지뷰 (Wikimedia REST) | 불필요 | '깊은 관심' 신호 |
| 빅카인즈 (한국언론진흥재단) | 선택 | 뉴스 표본 ~10배 확대 |
| Anthropic Claude | 필요 | AI 인사이트 리포트 |

### 키워드·경기 데이터 직접 수정하기

- **구단·선수 키워드** — `config.py`의 `TEAM_KEYWORDS`, `PLAYER_KEYWORDS`를 편집. 선수명을 추가하면 선수별 팬덤도 추적된다.
- **경기·중계 기록** — `data/match_records.csv`를 실제 WK리그 결과로 교체. 공식 사이트에 공개된 값이며, 한 번 입력하면 안 바뀐다.

## 분석 설계 노트

- **WK리그 단독 분석.** 다른 리그와 비교하지 않고 WK리그 자체에 집중한다.
- 네이버 검색 인덱스는 **상대값(0~100)**이라 절대 크기는 알 수 없다. 그래서 절대 규모 대신 **작년 동기 대비(YoY)·성장률·스파이크** 같은 변화 지표로 관심도를 판단한다.
- 검색 인구통계는 '실제 관람객'이 아니라 '검색으로 드러난 관심층'이다. 이들을 실제 팬으로 전환하는 것이 마케팅 과제이며, 대시보드는 그 출발점인 '관심층 측정'을 담당한다.

## 공모전 활용 가이드

제안서의 "지속 가능한 운영 체계" 파트에 첨부:

> "단발성 캠페인이 아니라, 데이터 기반으로 관심층 변화를 추적하고 다음 타겟을 자동 도출하는 반복 가능한 시스템이 있어야 중장기 프로젝트가 된다."

연맹·구단의 내부 데이터(관중·티켓·SNS 인구통계)가 연동되면 '관심 → 유입 → 전환' 전체 퍼널로 확장 가능하다는 점을 함께 제안하면 차별점이 명확해진다.
